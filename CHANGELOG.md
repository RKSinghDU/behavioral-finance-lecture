## How to Read This Changelog

### Categories

- **Added** - New features, content, or files
- **Changed** - Changes to existing functionality or content
- **Deprecated** - Features that will be removed in future versions
- **Removed** - Features or content that have been removed
- **Fixed** - Bug fixes, typo corrections, broken links
- **Security** - Security-related changes

### Semantic Versioning

We use `MAJOR.MINOR.PATCH` versioning:

- **MAJOR** (1.x.x) - Incompatible changes, major restructuring
- **MINOR** (x.1.x) - New features, backward-compatible additions
- **PATCH** (x.x.1) - Bug fixes, typos, small improvements

## Contributing to Changelog

When submitting a PR that will be included in the next release:

1. Add your changes under `[Unreleased]` section
2. Use appropriate category (Added, Changed, Fixed, etc.)
3. Include brief description of the change
4. Reference issue/PR number if applicable

**Example:**
```markdown
### Added
- New bias example: GameStop short squeeze (2021) (#42)
- Spanish translation of README.md (#45)

### Fixed
- Corrected typo in Prospect Theory section (#38)
- Updated broken link to SEBI report (#40)
```
